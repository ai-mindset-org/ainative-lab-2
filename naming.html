naming convention

один паттерн для каждого файла. дата в конце, en dash перед датой, lowercase в описании. имя файла — операционная ось vault'a: на нём держится поиск, граф и любой агент. этот документ — единый источник правды для s3, ai-native sprint vol2.

section 01 формула single source of truth
{project} {type} description – yyyy-mm-dd.md
{project} {type} description – date

читай слева направо: project отвечает на «к чему это», type — «что это», description — «о чём именно», date — «когда зафиксировано». два первых блока в фигурных скобках, описание lowercase свободным текстом, дата всегда последней через en dash.

важно: en dash (–, U+2013), не дефис (-) и не em dash (—). на mac — option + -. макс длина 80 символов.

не делай так: дата в середине ломает сортировку и поиск. {s3} {2026-05-06} {workshop} ... — нельзя. {s3} {workshop} ... – 2026-05-06.md — да.

section 04 структура vault where the file lives

имя описывает «что» — папка описывает «где». ниже — каноническая структура командного vault'a (ai-mindset-2026/). файл должен лежать в правильной папке по своему типу и проекту, имя — лишь дополняет иерархию.

ai-mindset-2026/ ├── AGENTS.md # индекс для ai-агентов (читать первым) │ ├── aim-rules/ # {rule}, {prd}, {tool}, {prompt}, {skill} │ ├── rules_tech/ # технические стандарты (naming, sync, ...) │ ├── rules_texts/ # стандарты коммуникации │ ├── rules_visuals/ # визуальные стандарты │ └── skills/ # командные claude-code скиллы │ ├── inbox/ # {research}, {analysis}, {overview}, черновики │ └── logs/ # логи изменений и daily updates │ ├── meetings/ # {meeting}, {transcript}, {summary} │ ├── Labs/ # все лаборатории (s3, x26, f25, w26, auto*) │ ├── s3 – ai-native vol2/ │ │ ├── s3-artifacts/ │ │ ├── s3-participants/ # {s3} {profile} @user - Name.md │ │ ├── s3-results/ # {s3} {feedback}, {s3} {retro} │ │ └── s3-sources/ # {s3} {transcript}, {s3} {summary} │ └── _inner-labs/ # общая инфраструктура лаб │ ├── Org/ # организация │ ├── HR/ │ │ └── roles/ # {role} Name.md │ ├── Infrastructure/ │ ├── Marketing&Sales/ │ ├── Partners/ # партнёрские карточки │ └── Processes/ # {process} название – дата.md │ ├── Products Ecosystem/ │ ├── Space/ # сообщество для фаундеров │ ├── Circle/ # круг поддержки │ ├── B2B/ # корпоративные клиенты │ └── Consulting/ # консалтинг │ ├── aim context/ # контексты для агентов ├── aim-guides/ # гайды └── _data/ # медиа, аналитика
правило vault: aim-rules/ — только для официальных правил, prd, шаблонов. для черновиков, ресерчей, обзоров — inbox/. не создавай новые папки в корне без обсуждения.
section 05 yaml frontmatter metadata layer

frontmatter — это метаслой имени. имя файла — для глаз, frontmatter — для dataview, поиска и агентов. минимум: tags, type, created.

--- tags: - lab/s3 - type/research - status/draft type: research project: s3 created: 2026-05-06 status: draft author: - "[[{role} Alexander Povaliaev]]" linear: AIM-2925 ---
keyназначение
tagsмассив тегов (см. section 06) для фильтрации в obsidian
typeдублирует {type} из имени без скобок — для dataview-запросов
createdдата создания, всегда yyyy-mm-dd
statusdraft / review / final / archived — жизненный цикл
section 06 теги classification axis

теги — вторая ось имени. через них собираются срезы, которые папкой не описать (например, все «active drafts» из разных лаб). держи 5 канонических семейств ниже, не плоди синонимов.

familyпримерназначение
lab/{cohort}lab/s3привязка к лаборатории — главный фильтр участника
type/{type}type/researchдублирует {type} имени для tag-based search
status/{state}status/draftactive / draft / archive — жизненный цикл
format/{kind}format/transcriptформат содержимого: transcript, summary, scan
project/{name}project/mcrmсквозной проект через несколько лаб
section 07 setup в obsidian first 5 minutes
00

поставь quickadd

community plugin quickadd — для шаблонов имени по горячей клавише. settings → community plugins → browse → quickadd → install + enable. без него ты будешь набирать {s3} {type} ... – 2026-05-06.md руками каждый раз.

01

создай структуру папок

скопируй дерево из section 04 в свой vault. для участника s3 минимум: aim-rules/, inbox/, meetings/, Labs/s3/. остальное — по мере появления контента.

02

дефолт для новых заметок — inbox/

settings → files & links → default location for new notes → in folder specified below → inbox/. всё новое падает в один буфер. оттуда — переименовываешь по конвенции и переносишь в нужную папку.

03

dataview для фильтрации по frontmatter

community plugin dataview → install + enable. теперь можно собирать дашборды по yaml-полям: FROM #lab/s3 WHERE status = "draft" — все черновики по s3.

section 09 track-коды (когорт) cohort sub-streams

track — тематический поток внутри лаборатории. используется только если в когорте явное разделение на параллельные потоки. формат: {cohort} {track} {type} название – дата.md

trackописание
{main}основной поток лаборатории
{art}ai + творчество, креативные практики
{vibecoding}vibe coding, разработка с llm
{coaching}коучинговый поток
{ethics}этика ai, философия
{automation}автоматизация, n8n, agent flows
не злоупотребляй: в s3 один основной поток, треки не нужны. их добавляют только когда лаборатория явно разделена (например f25 имела 7 параллельных треков).
section 10 исключения no curly braces

три файла существуют без фигурных скобок и без даты в имени — для совместимости с инструментами и github.

AGENTS.md — индекс директории для ai-агентов. читается первым при заходе агента в папку.

README.md — для github-совместимости. внутри — ссылка на agents.md.

CHANGELOG.md — история изменений (опционально, для проектов с релизами).

также без скобок: файлы инструкций для отдельных агентов — CLAUDE.md, OPENAI.md, CURSOR.md.
