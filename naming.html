## naming convention (для следующего файла)

формула: `{project} {type} description – yyyy-mm-dd.md`

| код | использовать когда |
|-----|---------------------|
| `{AIM}` | артефакты команды AI Mindset |
| `{s3}` | этот sprint, моя личная заметка по нему |
| `{ai-native}` | концепт-теги: AI-native, agentic, harness |
| `{pos}` | personal os артефакты |

типы: `rule`, `prd`, `skill`, `prompt`, `tool`, `template`, `research`, `analysis`, `overview`, `meeting`, `transcript`, `program`, `material`, `brief`, `playbook`, `evidence`, `announce`

правила:
- дата всегда в конце через en dash `–` (не em dash `—`, не дефис `-`)
- lowercase в описании, proper nouns ok
- 80 символов максимум
- никаких суффиксов интерфейса (`– claude code`, `– cursor` deprecated с 2026-02-20)

02как пользоваться

1

скопируй шаблон выше

нажми кнопку скопировать весь файл, шаблон уйдёт в буфер целиком — frontmatter, тело и naming-rule footer.

2

сохрани под именем по формуле

формула: {project} {type} description – yyyy-mm-dd.md. дату пиши через en dash –, не em dash — и не дефис -.

3

заполни блоки сверху вниз

контекст → сила → workflow → граница → passport → evidence → review. блоки можно оставлять пустыми, но лучше пройти все — каждый ловит свой слой системы.

4

повтори через неделю

вернись к review-date, сравни с этим документом — что изменилось, какие блоки наполнились, какой workflow реально живёт.
